## プロジェクト概要

3Dプリンターをペンプロッターとして使用するための、2Dベクター画像からGコードを生成するデスクトップアプリケーション。
ペンはプリンターノズルにオフセットして取り付ける前提。

- **参照プロジェクト**: https://github.com/Xenoah/Kasynel-Slicer (FDM スライサー / PyQt6 / Python)
- **リポジトリ**: c:\GitHub\Penplot-Gcoder

## 仕様詳細

### Machine タブ — プリンター設定

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| プリンタープロファイル | カスタム | Ender-3 / Prusa MK3 / カスタム |
| ファームウェア | Marlin | Marlin / Klipper |
| ベッドサイズ X | 220mm | 描画可能エリア幅 (=用紙幅の上限) |
| ベッドサイズ Y | 220mm | 描画可能エリア奥行き (=用紙高さの上限) |
| 原点位置 | 左下 | ベッド上の基準点 (左下 / 中央) |
| 有効描画エリア | 自動計算・読み取り専用 | ペンオフセットを差し引いた実効エリア |

**用紙サイズはベッドサイズで決まる。** 別途「用紙サイズ」設定はない。

#### 有効描画エリアの計算式

```
X_min = max(0,      -offset_x)
X_max = min(bed_x,  bed_x - offset_x)
Y_min = max(0,      -offset_y)
Y_max = min(bed_y,  bed_y - offset_y)
有効幅  = X_max - X_min
有効高さ = Y_max - Y_min
```

- オフセット変更時にリアルタイムで再計算・表示更新
- 配置がはみ出す場合は警告バナー表示・Gコード生成をブロック
- プレビュー: ベッド枠（グレー）と有効描画エリア枠（緑）を区別表示

### Pen タブ — ペン設定

#### ペン動作

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| ペン下げ Z 高さ | 0.0mm | 紙に接触する Z 座標 |
| ペン上げ Z 高さ | 5.0mm | 移動時のリフト量 |
| ペン下げ速度 | 300 mm/min | Z 軸の下降速度 |
| ペン上げ速度 | 1000 mm/min | Z 軸の上昇速度 |
| タッチダウン後ディレイ | 0ms | ペン下げ完了後・描画開始前の待機時間 (G4 P)。バウンド収束に使用 |
| リフトアップ前ディレイ | 0ms | ペン上げ直前の待機時間 (G4 P)。筆跡乾き待ちにも使用可 |

#### ペンオフセット（ノズルからの取り付けオフセット）

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| オフセット X | 0.0mm | ノズル基準でのペン先X方向ずれ |
| オフセット Y | 0.0mm | ノズル基準でのペン先Y方向ずれ |
| オフセット Z | 0.0mm | ノズル基準でのペン先Z方向ずれ |

- 生成Gコードの全XYZ座標にオフセットを自動加算
- プレビューはオフセット後の実座標で表示
- オフセット変更時に有効描画エリアを連動して再計算

### Fill タブ — 塗りつぶし設定

SVGの塗りつぶし領域やクローズドパスに対してハッチングパターンを生成する。

#### 基本設定

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| 塗りつぶし有効 | 無効 | 塗りつぶしを生成するか |
| 対象 | 塗りつぶし色あり | すべてのクローズドパス / 塗りつぶし色ありのみ |
| レイヤー順 | 輪郭→塗りつぶし | 輪郭先 / 塗りつぶし先 / 塗りつぶしのみ |

#### パターン種別

| パターン名 | 概要 |
|-----------|------|
| Lines | 平行ライン |
| Cross Hatch | 2方向クロスハッチング |
| Tri Hatch | 3方向 (0°/60°/120°) ハッチング |
| Contour Offset | 輪郭を内側に等間隔オフセット (同心状) |
| Zigzag | ジグザグ往復ライン |
| Dots | 等間隔の点描 |
| Spiral | 輪郭から内側へのスパイラル (円形領域向け) |

#### パターン共通パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| 線間隔 | 1.0mm | ハッチングラインの間隔 |
| 角度 | 45° | ライン方向の角度 (Lines / Zigzag 用) |
| 輪郭との余白 | 0.0mm | 塗りつぶし開始位置を輪郭から何mm内側にするか |
| パス最適化 | 有効 | 塗りつぶしパスも移動距離最小化 |

#### Cross Hatch 追加パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| 第2角度オフセット | 90° | 2本目のライン方向 (第1角度 + オフセット) |

#### Contour Offset 追加パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| 最内ライン処理 | そのまま | そのまま / 省略 / 点描 |

#### Dots 追加パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|----------|------|
| 点の配置 | グリッド | グリッド / 千鳥 |
| 点の間隔 Y | 1.0mm | Y方向の間隔 (X方向は「線間隔」を使用) |

### GCode タブ — Gコードカスタマイズ

#### 開始コード (Start G-code)

テキストエディタで自由に編集可能。

```gcode
; デフォルト
G28
G90
G21
G1 Z{pen_up_z} F{pen_up_speed}
```

#### 終了コード (End G-code)

```gcode
; デフォルト
G1 Z{pen_up_z} F{pen_up_speed}
G1 X0 Y0 F{travel_speed}
M84
```

#### ペン上げ/下げコード (カスタマイズ可)

| 項目 | デフォルト |
|-----|----------|
| ペン上げコード | `G1 Z{pen_up_z} F{pen_up_speed}` |
| ペン下げコード | `G1 Z{pen_down_z} F{pen_down_speed}` |

サーボや専用ピンを使う場合にここを差し替える。

#### 利用可能なプレースホルダー

| 変数 | 内容 |
|-----|------|
| `{pen_down_z}` | ペン下げZ高さ |
| `{pen_up_z}` | ペン上げZ高さ |
| `{pen_down_speed}` | ペン下げ速度 |
| `{pen_up_speed}` | ペン上げ速度 |
| `{draw_speed}` | 描画速度 |
| `{travel_speed}` | 移動速度 |
| `{offset_x}` / `{offset_y}` / `{offset_z}` | ペンオフセット |
| `{bed_x}` / `{bed_y}` | ベッドサイズ |
| `{touchdown_delay}` | タッチダウン後ディレイ (ms) |
| `{liftup_delay}` | リフトアップ前ディレイ (ms) |

## 生成Gコードの構造

```gcode
; === Penplot-Gcoder ===
; 入力: example.svg
; 生成: 2026-03-06
; ベッド: 220x220mm  ペンオフセット: X=15.0 Y=0.0 Z=0.0
; 有効描画エリア: 205x220mm
; パス数: 48 / 推定描画時間: 12m34s

; ---- START G-CODE ----
G28
G90
G21
G1 Z5.0 F1000

; ---- DRAW ----
; パス 1/48
G1 Z5.0 F1000
G1 X25.0 Y20.0 F6000        ; オフセット加算済み (元: X10.0 Y20.0)
G1 Z0.0 F300                 ; ペン下げ
G4 P50                       ; タッチダウン後ディレイ 50ms
G1 X65.0 Y20.0 F1500         ; 描画
G4 P0                        ; リフトアップ前ディレイ
G1 Z5.0 F1000                ; ペン上げ
...

; ---- END G-CODE ----
G1 Z5.0 F1000
G1 X0 Y0 F6000
M84
```

## SVG複数色の扱い

- SVGの各パスの `stroke` / `fill` 色を読み取り、色ごとにグループ化してパスリストに表示
- ユーザーはパスリストで各グループの描画順序を並び替え可能
- 色グループ = 「ペン交換タイミング」として扱い、グループ間に `M0 (カスタム一時停止コード)` を挿入するオプションを追加

## Undo/Redo

- 配置・スケール・回転などの操作に対して Undo/Redo (Ctrl+Z / Ctrl+Y) をサポート
- PyQt6 の QUndoStack を使用
